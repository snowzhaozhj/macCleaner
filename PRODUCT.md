# Product

## Register

product

## Users

**主要用户：Mac 开发者。** 磁盘被 Xcode DerivedData、node_modules、Docker 镜像、brew 缓存等开发产物持续占满。使用场景通常是终端里、磁盘告警时或例行维护——他们熟练使用命令行工具，追求速度，但对"删错东西"零容忍。要完成的任务：快速看清"谁在占空间"，安全地释放它，且全程可审查。

**次要用户：普通 Mac 用户。** 磁盘告警时想找到并清理占空间的东西，但不愿付费、不愿装一个比问题更大的工具。他们需要界面自己解释自己，尤其需要清楚知道"删这个安不安全"。

## Product Purpose

macCleaner 是一个完全开源、零遥测、零订阅的 Mac 清理工具，通过 CLI / TUI /（未来）GUI 三层界面共享同一清理引擎。它的存在是为了回答一个问题——"怎样干净地清理垃圾，而不搞坏系统"。成功 = 用户信任它到愿意定期使用：扫描快（目标 < 30s）、每个将删项及预估空间一目了然、无静默删除、危险项永远显式二次确认。

## Brand Personality

**专业、冷静、工程感。** 三词：clear / trustworthy / fast。谱系是 Linear、btop、lazygit——终端原生美学，信息密集但不喧闹，靠克制赢得信任。语气是同行工程师之间的直白，不讨好、不吓唬。界面应"消失在任务里"：用户注意的是自己的磁盘，不是我们的装饰。情绪目标是**放心**——每一个删除动作前，用户都确切知道会发生什么、能否恢复。

## Anti-references

- **CleanMyMac 式恐吓营销**（明确反例）：红色告警轰炸、夸大的"危险/立即清理"、进度动画表演式拖延、逼迫付费的暗黑模式。我们用红色**只**标真实的不可恢复风险，不制造焦虑。
- 花哨渐变 / 彩色堆砌 / emoji 满屏——削弱工具可信度。
- 装饰性动画、表演式加载序列。进度只传达状态，不表演。

## Design Principles

1. **危险是稀缺信号，不是营销手段。** 红色只留给真正不可恢复的操作（Risky 项、删除确认）。滥用告警色 = 变成我们反对的竞品。安全等级驱动配色，配色不驱动情绪。
2. **永不把颜色当唯一信息通道。** 安全等级永远同时用 颜色 + 形状（●▲✕）+ 文字标签 三通道编码。这是 NO_COLOR、色盲、以及未来任何渲染后端都必须守住的底线。
3. **无静默删除，删除即预览。** 任何删除动作前列出完整路径清单；Risky 项默认不勾选且需 type-to-confirm。信任来自可审查，不来自承诺。
4. **密度服务熟练，清晰服务新手。** 一屏内尽可能多的有效信息（开发者要速度），但每个符号/颜色/标签都能自解释（普通用户要看得懂）。不为好看牺牲密度，也不为密度牺牲可读。
5. **一套语义，多个后端。** 视觉决策以框架无关的语义角色表达（safe / moderate / risky / accent / ink / muted…），而非绑定终端 ANSI 或某个 GUI 框架。引擎已经三层复用，视觉语言也应如此。详见 DESIGN.md。

## Accessibility & Inclusion

- **不依赖颜色的三通道编码**（颜色 + 形状字形 + 文字标签）已是核心不变量，任何界面/后端都须保留。
- **NO_COLOR 支持**（遵循 no-color.org）：无色环境下前景回退终端默认色，光标高亮改用反显。
- **色盲友好**：●▲✕ 与"安全/中等/危险"标签独立于颜色可辨。
- 对比度：桌面端移植时，正文文字须 ≥ 4.5:1，muted 文字只承载次要信息且须验证对比度（见 DESIGN.md 对比度门槛）。
- 减少动效：桌面端所有动画须有 `prefers-reduced-motion` 降级（TUI 侧 spinner 已是最小动效）。

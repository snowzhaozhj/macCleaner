# 差异化在执行，不在与竞品对立或省略功能

## 反模式（曾犯）

在 GUI 重设计 ideation 里，把"磁盘空间总览 / 菜单栏 HUD"方向判"出局"，理由之一是**"照搬 Mole 付费版卖点，与'免费开源'定位矛盾"**、以及潜在的"要和别人不一样"。用户当场戳破：

> "我们不是要抄袭别人，但总不能大家都有的功能我们没有吧，然后原因还是因为要和别人不一样，这个结论根本站不住脚。"

这条推理是**反的**。它把差异化理解成"少做一个别人都有的功能"或"凡竞品有的就要回避"。

## 正确原则

**差异化是把同样的（甚至预期内的）功能做得更好，不是省略它、也不是刻意与竞品对立。**

- **品类基本盘功能**（如磁盘空间总览、菜单栏概览、卸载残留清理——DaisyDisk/CleanMyMac/macOS 储存空间都有）**缺了是短板，不是优点**。
- macCleaner 的差异化维度（见 `STRATEGY.md` "Our approach"）是**执行质量**：免费、开源、零遥测、诚实（不恐吓）、快、轻、安全可审查。
- **"竞品有"既不是做的理由，也不是不做的理由。** 判断只看：这个功能是否服务我们的用户与方向（`STRATEGY.md` 的 target problem / who it's for / tracks）。
- **"beat-mole = 免费直击其付费点"** 的正确含义是：把 Mole 收费的能力**免费送出去**正是打法——所以"照搬竞品付费功能"若指"我们也免费提供该能力"，是**支持理由**，不是矛盾。

## 合法的拒绝/推迟理由（对照）

砍或推一个功能，理由必须落在**功能自身与我们方向的关系**上，不是"竞品有/要不一样"：

- **工程面/范围**："它碰 app 生命周期/常驻进程，超出本轮呈现层范围" → 推迟到独立一轮，不是否决。
- **明文约束**：与 `STRATEGY.md` 已写的承诺冲突（如"不成为系统负担"）——但要分清约束的是**怎么实现**（常驻要轻量、无角标无通知无高频后台扫描）还是**做不做**。多数情况约束的是前者。
- **用户价值弱/优先级低**：对目标用户价值不足，排后。
- **安全底线**：违反不可退让的安全语义（Risky 永不预选、type-to-confirm、默认 Trash）——这才是真红线。

## 如何应用

- ideation/brainstorm/plan 里给任何方向下裁决前，检查理由是否是"竞品有/要不一样"——若是，作废重判。
- 竞品可以作为**参考与执行标尺**（"DaisyDisk 的空间图很直观，我们要做得同样好且免费"），不能作为**做不做的判据**。
- 我们朝**自己的方向**（STRATEGY）演进，与竞品的关系是"同类做得更好"，不是"为不同而不同"。
